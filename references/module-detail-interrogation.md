# 模块详设拷问

当 PM Assistant 进入 `S5.6`，或任一模块需要写入 `modules/<序号>-<模块名>.md` 时读取本文件。

目标：
- 将主 PRD 的模块总览下沉为可执行的真实颗粒度模块详设。
- 防止模块只写“功能说明”，漏掉入口、主客态、状态机、跳转、文案、数据、风控、埋点和验收。
- 为每个需要模块文件的模块形成闭环。

## 触发条件

以下任一条件命中时必须进入 `S5.6`：
- 一个版本包含 2 个及以上核心功能模块。
- 任一模块涉及已有产品迭代、多入口、多角色、主客态、跨端跳转、邀请、分享、授权、支付、积分、审核、异步任务、消息通知、弹窗优先级、旧数据兼容或灰度发布。
- 用户要求“颗粒度模块化文档 / 模块详设 / 像真实 PRD 一样细”。

## 逐模块拷问清单

| 序号 | 拷问维度 | 必须问清 |
|------|----------|----------|
| 1 | 模块定位 | 对应哪个主 PRD 目标？是否 MVP 必须项？砍掉后主流程是否还能跑？依赖哪些模块？ |
| 2 | 环境继承与覆盖 | 继承哪些全局环境？本模块额外有哪些端、业务、技术、合规、发布约束？ |
| 3 | 现有逻辑 | 原页面 / 原流程 / 原数据是什么？本期新增、修改、删除、复用和兼容风险是什么？ |
| 4 | 入口矩阵 | 所有入口位置、展示条件、隐藏条件、点击行为、未登录 / 无权限 / 未安装 / 版本不足处理、完成后回流是什么？ |
| 5 | 角色 / 主客态 | 发起方、接收方、被影响方、第三方访问者分别能看见什么、能做什么、状态如何变化？ |
| 6 | 状态机 | 关键业务对象的初始、成功、失败、过期、撤销、重试、并发优先级和下一状态是什么？ |
| 7 | 页面线框 | 页面 / 弹层 / 卡片的信息层级、主 CTA、次操作、空态、失败态、风险提示是什么？ |
| 8 | 跳转与回流 | 来源、动作、前置状态、目标页面、中间弹窗优先级、成功回流、失败或取消处理是什么？ |
| 9 | 文案资产 | 标题、副标题、按钮、Toast、空态、错误提示、分享卡片、动态变量、字数限制和截断规则是什么？ |
| 10 | 数据 / 接口 / 埋点 | 字段、创建和维护方、接口输入输出、失败码、埋点事件、指标、验收用例是什么？ |

## 完成门禁

- 每个模块都有稳定模块 ID，并能追溯到主 PRD 的模块总览。
- 每个模块至少有一条主路径验收用例和一条异常路径验收用例。
- 复杂模块必须有业务对象状态机；若没有状态机，必须说明为什么不需要。
- 所有入口、跳转、文案、数据字段和埋点必须能追溯到具体场景或验收标准。
- 模块环境覆盖项不得与主 PRD 全局环境冲突；冲突必须标记为待确认。

## 阶段闭合输出

```markdown
## 阶段闭合：S5.6 模块详设拷问

### 已确认
- 模块清单与优先级：
- 每个模块的详设完整度：完整 / 部分完整 / 阻塞
- 每个模块需要写入的 modules/<序号>-<模块名>.md：

### 未确认
- 无 / 列出阻塞项

### 推荐答案
- 对未确认项给出推荐答案

### 下一步
- 进入 S6 / 停在 S5.6
```
