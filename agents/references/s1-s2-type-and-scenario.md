# S1-S2 产品类型分流与场景追问树

本文件供 SKILL.md S1 和 S2 阶段引用。

## S1 类型自动推断规则

当用户未明确指定产品类型时，按以下规则推断：

- 含"用户 / 会员 / 积分 / 商城 / 社交 / 游戏" → B2C
- 含"企业 / SaaS / CRM / 后台管理 / 客户管理" → B2B
- 含"内部 / 管理系统 / 工单 / 效率工具 / 运营后台" → 内部工具
- 含"平台 / 市场 / 撮合 / 双边 / 交易市场" → 平台型
- 跨多个类型 → 混合型

## 不同类型的 PRD 侧重差异

| 产品类型 | PRD 侧重模块 | 关键差异 |
|----------|--------------|----------|
| B2C | 用户旅程、增长指标、A/B 测试方案 | 重交互、重数据、多写用户旅程 |
| B2B | 权限模型、多租户、SLA、集成接口 | 重功能完备性、重安全合规、多写 API 对接 |
| 内部工具 | 操作效率、与现有系统集成、培训成本 | 重实用性、轻视觉、多写操作流程 |
| 平台型 | 多角色交互、供需匹配、生态规则 | 重角色拆分、重规则引擎、多写各角色视角 |

## S2 通用追问

- 主要用户是谁，次要用户是谁
- 在什么场景下使用
- 使用前用户遇到什么阻碍
- 使用后什么结果算成功

## B2C 加问

- 用户动机是什么
- 首次使用触发点是什么
- 核心转化动作是什么
- 留存理由是什么
- 产品价值偏效率、情绪、内容、关系、交易还是工具

## B2B 加问

- 使用角色、管理角色、采购或决策角色分别是谁
- 当前业务流程怎么跑
- 哪些步骤最耗时、最容易错或最难追踪
- 权限、审批、协作和组织约束是什么

## 内部工具加问

- 操作者每天使用频率和单次操作时长
- 当前用什么方式完成同一任务（Excel / 邮件 / 手动 / 旧系统）
- 操作流程中最容易出错或最耗时的环节
- 需要与哪些现有系统交互（CRM / ERP / 工单 / 通讯）
- 新手多久能独立操作，培训成本预估

## 平台型加问

- 供方、需方和平台运营三个角色的诉求分别是什么
- 供方为什么要来平台，需方为什么要在平台交易
- 供需匹配规则是什么（搜索排序 / 推荐算法 / 筛选条件）
- 平台佣金 / 抽成 / 结算规则是什么
- 任一侧冷启动时另一侧是否有价值

## 混合型加问

- 各侧用户分别获得什么价值
- 各侧价值是否互相依赖
- 哪一侧是 MVP 的优先启动侧
- 各侧的 PRD 侧重差异是什么
